1. OVERZICHT & TECHNOLOGIE
================================================================================

Het lpbunified-master project is een Belgisch gemeentelijk loonverwerkingssysteem
(LPB = LoonPakket Beheer), ontwikkeld door Schaubroeck. Het beheert de lonen
van gemeentepersoneel en mandatarissen.

Technologiestapel:
  - Java (Spring Boot) + JPA/Hibernate : backend business logic
  - C/C++ (NLP module)                 : legacy rekenengine
  - AngularJS (lima-ria)               : frontend
  - TypeScript / Angular (lima-ria2)   : moderne frontend
  - MySQL / Informix                   : databases (BI datamarts)
  - JAX-RS REST API                    : web services

Terminologie in het project:
  - "Wedde"        = salaris / loon
  - "Schaal"       = loonschaal (barema)
  - "Weddeschaal"  = loonschaal (synoniem)
  - "Trap"         = salaristrap / trede binnen de schaal
  - "Versie"       = schaalversie (gekoppeld aan CAO of akkoord)
  - "Ritme"        = aanscienniteitritme (tempo van trapverhoging)
  - "Internnr"     = intern nummer van de loonschaal
  - "Schaalnummer" = extern, leesbaar schaalnummer
  - "GEA"          = Geldelijke Anciënniteit
  - "SCA"          = Schaal-Anciënniteit
  - "DIA"          = Dienstanciënniteit
  - "NIA"          = Niveau-Anciënniteit
  - "GRA"          = Graad-Anciënniteit
  - "SOGID"        = Schaal Overgangsgroep ID

2. KERNCONCEPT: SCHAAL / WEDDESCHAAL
================================================================================

Een loonschaal in dit systeem bestaat uit:

  A. SchaalDefinitie
     - Internnummer   : unieke interne code (bv. "A1/1")
     - Schaalnummer   : leesbaar schaalnummer
     - Soort          : type schaal
     - Grpkls         : groepklasse
     - SchaalNiveau   : niveau-aanduiding (bv. A, B, C, D, E)
     - RszppoNiveau   : RSZ-PPO niveau (sociaalzekerheidsklassificatie)
     - InschakelInternnr : bij promotie te gebruiken volgschaal
     - Rang           : rang binnen de loonschaal

  B. SchaalVersie (per internnr + versie)
     - MinBedrag      : minimumbedrag van de schaal (basisbedrag)
     - MaxBedrag      : maximumbedrag van de schaal
     - BevordGrd      : bevorderingsgraad
     - BevordNr       : bevordering schaalnummer
     - AanvangNr      : aanvangsschaalnummer
     - BevordMin      : minimumbedrag bij bevordering
     - AanvangMin     : minimumbedrag bij aanvang
     - Grensverhoging : grensverhogingsbedrag
     - EuroOriVersie  : euro-oorsprongsversie
     - EuroCvt        : euro-conversie indicator

  C. SchaalVerhoging (trappenstructuur per schaal+versie)
     - Lijn    : lijnnummer (volgorde van verhogingsregel)
     - Aantal  : aantal maal deze verhogingsregel van toepassing is
     - Periode : aantal tijdsperiodes (maanden/jaren) per trap
     - Bedrag  : verhogingsbedrag per trap

  Berekening loonbedrag voor trap N:
    startbedrag = schaalversie.minBedrag
    voor elke verhogingsregel (lijn):
      voor elk "aantal":
        na "periode" maanden → bedrag += verhoging.bedrag
    resultaat = bedrag na N trappen

  D. WeddeBepaling (per personeelslid/opdracht)
     - Graad              : aangestelde graad
     - PersoneelsGroep    : personeelsgroepsclassificatie
     - DienstStaat        : dienststaatnummer
     - GEA ritme/maanden/datum : geldelijke anciënniteitsparameters
     - SCA ritme/maanden/datum : schaalanciènniteitsparameters
     - DIA ritme/maanden/datum : dienstanciènniteitsparameters
     - NIA ritme/maanden/datum : niveauanciènniteitsparameters
     - GRA ritme/maanden/datum : graadanciènniteitsparameters
     - BevorderingsExamen : indicator bevorderingsexamen
     - InschakelSchaal    : inschakeling nieuwe schaal
     - PrincipeFunctioneleLoopbaan : 0/1 vlag

3. JAVA INTERFACES (lpb-tools-api)
================================================================================

Bestandslocatie: lpb-tools/lpb-tools-api/src/main/java/be/schaubroeck/lpb/tools/

Kerninterfaces:

  - SchaalDefinitie.java
    Beschrijft de definitie van een loonschaal (internnr, schaalnr, niveau, rang).

  - SchaalVersie.java
    Eén specifieke versie van een schaal: min/max-bedrag, bevorderingsinfo.

  - SchaalVerhoging.java
    Eén lijn in de trappenstructuur: lijn, aantal, periode, bedrag.

  - WeddeBepaling.java
    Interface voor de wedde-toewijzing aan een opdracht (personeelslid):
    alle anciënniteitstypes, graad, dienststaatinformatie, schaalovergang.

  - WeddeBerekening.java
    Interface voor de berekeningsparameters: rekenwijze, jaar/maand/dag-omzetting,
    RSZ-dagenstelsel, vakantiegeld-regeling, eindejaarstoelage-regeling,
    prestatieverhouding, arbeidscyclus.

  - SchaalKoppeling.java
    Koppeling tussen de interne NLP-schaal en de PDOS/CAPELO-schaal.
    Bevat: status, admin, KBO-nummer, internnr, versie, klasse, beslissingsdatum.

  - CapeloWeddeschaal.java
    Externe schaalreferentie voor de CAPELO-pensioenkadasterapplicatie.
    Bevat: capeloWerkgever, referentie, startDatum, eindDatum.

  - MandatarisBarema.java
    Barema specifiek voor mandatarissen (burgemeester, schepenen, raadsleden).
    Bevat: begin/eindatum normale pensioenberekening, klasseverheffing.

  - HrWedde.java (in jpa/cl/)
    HR-loongegevens voor CoreHR-integratie.

9. LOONSIMULATOR
================================================================================

Locatie: spring/lima/lima-server/src/main/java/be/schaubroeck/lpb/loonsimulator/

De loonsimulator berekent het nettoloon op basis van de loonschaal.

Klasse: Weddeschaal.java
  Methoden:
  - berekenTrap(internnr, trap, service)
    → Berekent het bruto maandbedrag voor een bepaalde trap
    → Gebruikt SchaalVersie (minBedrag) + SchaalVerhogingen (trappenstructuur)

  - berekenTrap(schaalversie, schaalVerhogingen, trap)
    → Kernalgoritme:
      1. Start met schaalversie.minBedrag
      2. Loop door alle SchaalVerhoging-lijnen
      3. Per lijn: "aantal" iteraties van "periode" maanden
      4. Na elke periode → bedrag += verhoging.bedrag (afgerond op 2 decimalen)
      5. Stop bij trap N

Simulatiecomponenten (Chain of Responsibility patroon):
  - SimuleerBrutoLoon.java           : bruto jaarwedde
  - SimuleerRSZBijdrage.java         : RSZ-bijdrage werknemer
  - SimuleerBijzondereBijdrageSZ.java : BBSZ
  - SimuleerBedrijfsvoorheffing.java : bedrijfsvoorheffing
  - SimuleerVlaamseKorting.java      : Vlaamse korting
  - SimuleerNettoLoon.java           : nettoloon

10. CAPELO / DMFA-INTEGRATIE
================================================================================

CAPELO = Carrièregegevens en Pensioenen van Lokale en Provinciale besturen
DMFA   = DéclarationMultifonctionnelle / Multifunctionele Aangifte (RSZ)

Concept: elke interne NLP-loonschaal moet worden gekoppeld aan een
officiële CAPELO/PDOS-schaaldefinitie zodat pensioenberekeningen correct zijn.

Klassen:
  - CapeloWeddeschaal       : externe CAPELO-schaaldefinitie
  - CapeloSalaryScale       : CAPELO salarisschaal
  - CapeloSalaryScaleDetail : detail (startdatum, trappenstructuur)
  - CapeloSchaalVerhoging   : CAPELO trapverhoging
  - SchaalKoppeling         : NLP-schaal ↔ CAPELO-schaal koppeling

Workflow:
  1. NLP-beheerder signaleert nieuwe/gewijzigde schaal → status NOG_TE_KOPPELEN
  2. DMFA-admin koppelt aan bestaande CAPELO-schaal OF vraagt nieuwe aan
  3. Aanvraag gaat naar PDOS (Pensioendienst voor de Overheidssector)
  4. PDOS geeft ticket terug → status TICKETNR_ONTVANGEN
  5. PDOS verwerkt → geeft DMFA-code → status DMFACODE_ONTVANGEN
  6. Koppeling is volledig → status AUTOMATISCH_GEKOPPELD of bevestigd

PDF-rapporten:
  - Weddeschalen PDF : overzicht alle schaalkoppelingen (SchaalKoppelingPDF)
  - Uitwerking PDF   : trappenuitwerking van één schaal (UitwerkingPDF)

E. Bedrijfsvoorheffing (BV) berekening op basis van barema

   BerekenBvGewoon.java → methode berekenBVGewoneBezoldigingBarema():
   1. Bereken jaarbedrag (belastbaar × 12)
   2. Trek forfaitaire BV af (voor niet-gepensioneerden)
   3. Pas belastingschalen toe (oud = pré-2001 / nieuw = post-2001)
   4. Verminderingen voor gezinstoestand + kinderenTenLaste
   5. Pensioenvrijstelling + jobkorting + ZAO-vermindering
   6. Pas crisis-tax-coëfficiënt toe
   7. Deel door 12 → maandelijkse BV (afronding: vóór 2023 = naar beneden)

   BerekenBvAchterstal.java → methode berekenBVAchterstalBarema():
   - 18 inkomstengroepen met percentages (5% - 32%)
   - Kindervrijstelling op basis van jaarinkomen vs. grensVrijstelling
   - Formule: BV = (belastbaar − vrijstelling) × percentage × crisisTax

   - mandatarisBaremaDetail  : MandatarisBaremaDetail

   - verhogingspercentage    : opslag hoofdplaats
   - verhoogdeBasisjaarwedde : basisjaarwedde × opslag

   VerminderdeWedde.java (pensioenrechten bij verminderd inkomen):

   - omgezetteMaanden     = omzettingsverhouding × volledigeMaanden
