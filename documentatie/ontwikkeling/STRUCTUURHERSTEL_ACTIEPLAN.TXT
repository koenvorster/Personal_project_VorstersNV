## 🔴 PRIORITEIT 1 — ROOT-VERVUILING (doe dit eerst, 30 min)

De root van een Python/Next.js project moet enkel bevatten: `pyproject.toml`, `requirements.txt`, `docker-compose.yml`, `Makefile`, `README.md`, `.gitignore`, `Dockerfile.*`, `alembic*.ini`, `.env.example` en de projectmappen.

### 1.1 — Verplaats `setup_backend.py` → `scripts/`

```powershell
Move-Item setup_backend.py scripts\setup_backend.py
```

**Reden:** Het is een uitvoerbaar setup-script, niet een package-configuratie. Hoort thuis bij de andere scripts.

### 1.2 — Archiveer `create_gems.cjs` + bijbehorende Node.js bestanden

```powershell
# Maak archief-map aan
New-Item -ItemType Directory -Path scripts\gems -Force

# Verplaats bestanden
Move-Item create_gems.cjs scripts\gems\create_gems.cjs
Move-Item package.json scripts\gems\package.json
Move-Item package-lock.json scripts\gems\package-lock.json
```

**Reden:** `create_gems.cjs` is een eenmalig Gemini Gems setup-script dat Playwright gebruikt. Het heeft geen plaats in de root van een Python/Next.js project. De root `node_modules/` verdwijnt daarmee vanzelf zodra je `npm install` alleen in `scripts/gems/` uitvoert.

> ⚠️ **Opgelet:** Verwijder `node_modules/` in de root NIET handmatig vóór je de move doet — eerst committen, daarna `node_modules/` laten verdwijnen via `.gitignore` (staat er al in).

### 1.3 — Twee startup scripts rationaliseren

- `Start-VorstersNV.ps1` — nieuwer, actiever

**Actie:**
```powershell
# Verplaats het oudere script naar archief
Move-Item START_PHASE_1.ps1 scripts\archief\START_PHASE_1.ps1
```

`Start-VorstersNV.ps1` blijft in de root — dit is het enige "quick start" script voor Windows-gebruikers.
