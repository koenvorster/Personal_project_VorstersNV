1. REVISIE 6 VISIE
──────────────────
Revisie 5 bouwde de intelligentie-fabriek: alle AI-modules, RAG, compliance en
portal-API zijn aanwezig maar alleen op localhost bruikbaar. Revisie 6 maakt het
platform ÉCHT klantklaar: een beveiligde, multi-tenant portal die een klant zelf
kan openen, zijn project live ziet analyseren, het rapport ontvangt per email én
kan downloaden — terwijl de backend automatisch deployt naar Google Cloud Run bij
elke commit op `main`. Het doel is één betalende klant te kunnen bedienen vóór
einde Wave W11, met W12 als differentiatie-laag.

            Keycloak blijft voor admin via Docker Compose lokaal

6.2 MULTI-TENANT — MINIMALE REFACTOR AANPAK
────────────────────────────────────────────
Row-Level Security via applicatielaag (niet PostgreSQL RLS):

  ALTER TABLE client_projects ADD COLUMN tenant_id VARCHAR(255);
  UPDATE client_projects SET tenant_id = klant_email;
  CREATE INDEX ix_client_projects_tenant_id ON client_projects(tenant_id);

Elke portal API-call voegt WHERE tenant_id = jwt.email toe.
PostgreSQL RLS plannen voor Revisie 7 als defense-in-depth.
